Taming the Home Lab: Friendly Local Domains with UniFi + Pi-hole + NPM

June 28, 2025 · homelab

Every homelab hits the same wall eventually: a dozen services, each living at an IP-and-port combo nobody can remember. 192.168.1.40:8123 is not a name — it's a punishment. This weekend I finally gave everything a friendly local domain, so Home Assistant lives at home.lab.internal and Proxmox at pve.lab.internal, with real TLS and no port numbers in sight.

Three pieces do the work: Pi-hole answers local DNS, Nginx Proxy Manager routes names to services, and UniFi hands both out to every device on the network. Here's the whole setup.

Step 1: Wildcard DNS in Pi-hole

Instead of adding a record per service, point a wildcard at the box running Nginx Proxy Manager. On the Pi-hole host, one line in a dnsmasq config does it:

# /etc/dnsmasq.d/99-lab.conf
# Send every *.lab.internal lookup to the NPM box
address=/lab.internal/192.168.1.20

$ sudo systemctl restart pihole-FTL
$ dig +short home.lab.internal # should answer 192.168.1.20
192.168.1.20

Step 2: Nginx Proxy Manager routes the names

NPM runs in Docker and terminates TLS for everything behind it. The compose file is short:

# docker-compose.yml
services:
  npm:
    image: jc21/nginx-proxy-manager:latest
    restart: unless-stopped
    ports:
      - "80:80"
      - "443:443"
      - "81:81"   # admin UI
    volumes:
      - ./data:/data
      - ./letsencrypt:/etc/letsencrypt

Then add a proxy host per service in the admin UI: home.lab.internal → 192.168.1.40:8123, and so on. Ten minutes of clicking, done forever.

Step 3: Tell UniFi about it

In the UniFi console, set your network's DHCP DNS server to the Pi-hole's IP so every device — phones included — resolves the new names automatically. No per-device setup, which matters in a house with four kids' worth of iPads.

Gotchas

  • Use .internal (or your own registered domain), never .local — mDNS already owns that and iOS devices will fight you.
  • Pi-hole must be the only DNS server DHCP hands out, or clients will race a resolver that's never heard of your lab.
  • For green-padlock TLS on internal names, do a DNS-01 challenge in NPM against a real domain you own.

That's it — the lab finally answers to names a human can remember. If you get stuck, email me; this stuff is more fun with company.
